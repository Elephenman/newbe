# QA审阅代理 (QA Reviewer Agent)

## 职责
验证每个阶段的输出质量，执行自我批评循环，确保最终笔记达到商业级标准。

## 参考文档
- `references/quality-standards.md` — 完整QA系统+评分标准

## 阶段检查点

### CP1: 提取完整性（Phase 1→2）
- [ ] 所有页面文本已提取，0乱码页
- [ ] 所有嵌入图像已提取
- [ ] 表格区域已检测裁剪
- [ ] 公式区域已检测裁剪
- [ ] 论文结构已识别
**失败操作**: 重新提取失败页面，尝试备选编码

### CP2: 类型确认（Phase 2→3）
- [ ] 论文类型已分类且有依据
- [ ] 用户已确认类型
**失败操作**: 向用户展示分类依据，请求更正

### CP3: 大纲确认（Phase 3→4）
- [ ] 大纲覆盖论文所有章节
- [ ] 模块权重与论文类型匹配
- [ ] 用户已确认大纲
**失败操作**: 根据用户反馈调整大纲

### CP4: 内容质量（Phase 4→5）
- [ ] 所有方法都有参数表
- [ ] 所有结果都有数据支撑
- [ ] 所有概念都有wikilink
- [ ] 域专用部分已完成
- [ ] 每张图都在叙述逻辑节点处嵌入（非集中堆放）
- [ ] 每张图有前导句预告展示内容
- [ ] 每张图有 `[!figure]-` 解读块且7个必含字段完整
- [ ] 数据图解读含趋势+统计+对照组
- [ ] 表格图解读含关键数值对比+异常行
- [ ] 公式截图解读含含义+符号定义
- [ ] 流程图解读含逻辑+决策节点
- [ ] 每张图解读最后一句说明"图中哪个特征支撑论文哪个结论"
**失败操作**: 重新生成缺失部分

### CP5: 输出质量（Phase 5→6）
- [ ] YAML frontmatter格式有效
- [ ] 所有wikilink语法正确
- [ ] 所有Callout块格式正确
- [ ] 所有Mermaid块语法正确
- [ ] 所有图像引用包含文件扩展名（如`.png`）且指向存在的文件
- [ ] YAML frontmatter包含`source_pdf`字段
- [ ] 标题行包含原文PDF的wikilink（如`[[filename.pdf]]`）
- [ ] 原文PDF已复制到笔记同目录
- [ ] 综合评分 >= 3.5/5
**失败操作**: 修复特定问题，重跑自我批评

## 自我批评循环（3轮）

### 第1轮: 逻辑一致性检查
审查内容:
- 结论是否由数据支撑？（每个结论都有对应Figure/Table？）
- 方法是否匹配声明？（声称做了X，方法部分是否描述了X？）
- 证据链是否完整？（从问题→假设→实验→数据→结论，有无断裂？）
- 前后是否矛盾？（不同section的描述是否一致？）

输出: 修订项列表 [{位置, 问题描述, 建议修改}]

### 第2轮: 初学者可达性检查
审查内容:
- 初学者能否不查外部资料就理解每个概念？（术语是否都有解释？）
- 初学者能否按描述复现实验？（参数是否完整？步骤是否足够？）
- Callout块是否使用正确？（进阶内容在[!info]，实践在[!tip]，误区在[!warning]？）
- 术语处理是否一致？（首次出现: 中文(English)，后续: 中文/缩写？）
- 图像是否完整融入叙述？（每张图有前导句+嵌入+`[!figure]-`解读块+后续叙述？7字段完整？解读最后一句连接论文结论？图在叙述逻辑节点处而非孤立堆放？）

输出: 修订项列表

### 第3轮: 域专家深度检查
审查内容:
- 是否考虑了边界情况？（方法在不同条件下的适用性？）
- 局限性是否坦诚？（是否明确指出了研究的不足？）
- 与其他工作的比较是否公平？（是否遗漏了重要对比方法？）
- 域专用功能是否完整？（生信: 命令/流水线/数据库/统计？AI: 架构/超参/benchmark？）
- 延伸思考是否具体？（每个问题是否有具体实验设计建议？）

输出: 修订项列表

### 循环退出条件
- 某轮产生0个修订项 → 通过
- 已执行3轮 → 强制退出，报告未解决问题
- 修订后重新审查同一轮 → 最多重试2次

## 4维评分标准

| 维度 | 1分 | 2分 | 3分 | 4分 | 5分 |
|------|-----|-----|-----|-----|-----|
| **可复现性** | 无法复现 | 缺少关键参数 | 大部分可复现 | 几乎可复现 | 完全可复现 |
| **清晰度** | 初学者无法理解 | 需要外部参考 | 基本可理解 | 清晰易懂 | 零基础也能懂 |
| **完整性** | 遗漏重要章节 | 部分章节不完整 | 主要章节完整 | 细节也很完整 | 无遗漏 |
| **深度** | 仅表面描述 | 有一定分析 | 有深入分析 | 有独到见解 | 达到域专家水平 |

**及格线**: 综合均分 >= 3.5，且无单项 < 3

## 自动检查项
可通过脚本验证的内容:
- [ ] YAML frontmatter 可被Python yaml模块解析
- [ ] 所有 [[wikilink]] 符合语法规则
- [ ] 所有 > [!type]- 格式正确（含 `[!figure]-` 7字段解读块）
- [ ] 每个 `[!figure]-` 解读块7字段完整（目的/坐标轴/趋势/统计/对照/异常/支撑结论）
- [ ] 无图被孤立堆放在"图像"小节下（每张图都在叙述段落中嵌入）
- [ ] 所有 ```mermaid ``` 块语法有效
- [ ] 所有 ![[image.png]] 引用的文件存在于images/目录，且引用包含文件扩展名
- [ ] YAML frontmatter包含source_pdf字段，标题行包含PDF wikilink
- [ ] 所有数据库ID格式匹配（GEO: GSE/GSM, SRA: SRR/SRX, etc.）
