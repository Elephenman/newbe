# 完整质量保证体系 (Complete Quality Assurance System)

> 本文档定义了论文深度解读技能的全程质量控制规范，包括5个阶段检查点、三轮自我批评循环、四维评分标准、自动检查项，以及质量报告模板。

---

## 1. 五阶段检查点详细规范 (Five-Stage Checkpoint Detailed Specification)

### CP1: 提取完整性检查 (Extraction Completeness Check)

**检查时机**: PDF解析与信息提取完成后

#### 检查项列表 (Checklist)

- [ ] PDF元数据成功提取（标题、作者、期刊、年份）
- [ ] 摘要完整提取且无误
- [ ] 所有章节标题正确识别
- [ ] 正文内容成功提取（无遗漏段落）
- [ ] 参考文献列表完整（尤其是引用的关键文献）
- [ ] 图表标题/图注完整提取
- [ ] 补充材料（S.I.）信息已记录（即使未完整处理）
- [ ] 关键词/受控词表提取
- [ ] DOI/PMCID/URL等标识符提取
- [ ] 资金来源/资助信息提取

#### 门条件 (Gate Conditions)

所有检查项必须通过（全部打勾），方可进入CP2。

#### 失败操作 (Failure Handling)

| 失败类型 | 具体问题 | 修复步骤 |
|---------|---------|---------|
| 元数据缺失 | 标题/作者/年份缺失 | 尝试从PDF首尾重新提取；若仍缺失，标注"待人工确认" |
| 章节识别失败 | 章节标题未识别 | 使用备用解析器或手动标注章节边界 |
| 正文提取不完整 | 段落缺失 | 检查PDF扫描质量；对扫描版使用OCR增强 |
| 参考文献缺失 | 引用列表为空 | 启用参考文献专项提取模块 |
| 图表信息缺失 | 图表标题未识别 | 启用图像描述提取器 |

#### 超时处理

当CP1验证超过3次仍未通过时：
1. 暂停自动处理流程
2. 输出部分提取结果供人工审查
3. 生成提取问题报告，标注具体失败点
4. 等待人工干预或跳过该论文

---

### CP2: 类型确认检查 (Type Confirmation Check)

**检查时机**: 论文类型分类决策完成后

#### 检查项列表 (Checklist)

- [ ] 类型分类决策树已执行
- [ ] 主类型判定有明确依据
- [ ] 副类型（如有）已标注
- [ ] 混合类型处理策略已应用（如适用）
- [ ] 类型判定结果已在笔记中标注
- [ ] 权重分布已按类型配置调整
- [ ] 识别特征与论文实际特征匹配度>70%
- [ ] 重点模块已按类型配置强化
- [ ] 弱化模块已按类型配置精简
- [ ] Callout使用模式已按类型配置应用

#### 门条件 (Gate Conditions)

至少9/10项通过，其中"类型分类决策树已执行"和"主类型判定有明确依据"必须通过。

#### 失败操作 (Failure Handling)

| 失败类型 | 具体问题 | 修复步骤 |
|---------|---------|---------|
| 类型判定不确定 | 两个以上类型得分接近 | 回溯决策树各节点结果；参考作者背景和发表期刊 |
| 特征匹配度低 | 匹配度<70% | 重新评估论文特征；考虑混合类型可能 |
| 权重配置错误 | 权重加和≠100% | 重新计算并修正权重分布 |
| Callout配置错误 | Callout类型与内容不匹配 | 按类型配置表重新映射Callout |

#### 超时处理

当CP2验证超过3次仍未通过时：
1. 采用保守策略：选择论文特征最明显的单一类型
2. 记录类型判定不确定性
3. 在笔记中标注"类型待确认"
4. 继续处理但增加人工审查标记

---

### CP3: 大纲确认检查 (Outline Confirmation Check)

**检查时机**: 笔记大纲生成完成后，初稿撰写前

#### 检查项列表 (Checklist)

- [ ] 大纲包含所有必含模块（背景/方法/结果/结论/其他）
- [ ] 各模块内容概要已填写
- [ ] 模块顺序符合论文逻辑结构
- [ ] 关键信息点已在大纲中标注
- [ ] 需要展开的详细点已标记
- [ ] 图表引用位置已在大纲中规划
- [ ] Callout使用位置已规划
- [ ] Wikilink占位符已放置（关联其他笔记时）
- [ ] 估计各模块篇幅符合权重分布
- [ ] 大纲已通过自我审查：逻辑连贯性

#### 门条件 (Gate Conditions)

至少8/10项通过，其中"所有必含模块完整"和"逻辑连贯性"必须通过。

#### 失败操作 (Failure Handling)

| 失败类型 | 具体问题 | 修复步骤 |
|---------|---------|---------|
| 模块缺失 | 缺少必含模块 | 按论文实际内容补充对应模块 |
| 顺序混乱 | 模块顺序不符合逻辑 | 参照论文结构重新排列 |
| 内容空洞 | 模块概要过于简略 | 返回论文阅读相关章节提炼要点 |
| 篇幅失衡 | 模块篇幅与权重差异>20% | 调整大纲内容分配 |

#### 超时处理

当CP3验证超过3次仍未通过时：
1. 采用论文原有章节结构作为大纲框架
2. 仅做必要的内容充实调整
3. 标记"大纲简化模式"
4. 继续处理但降低篇幅要求

---

### CP4: 内容质量检查 (Content Quality Check)

**检查时机**: 笔记初稿完成后，进入自我批评循环前

#### 检查项列表 (Checklist)

- [ ] 各模块内容完整，无遗留占位符
- [ ] 核心发现已准确提炼
- [ ] 方法描述可支持基本理解
- [ ] 数据/统计信息准确无误
- [ ] 图表引用正确且有描述
- [ ] Callout使用恰当且格式正确
- [ ] 无事实性错误或矛盾表述
- [ ] 术语使用一致
- [ ] 引用标注完整（论文内引用+外部参考）
- [ ] 敏感信息（如伦理声明）已保留

#### 门条件 (Gate Conditions)

至少9/10项通过，其中"无事实性错误"和"核心发现准确"必须通过。

#### 失败操作 (Failure Handling)

| 失败类型 | 具体问题 | 修复步骤 |
|---------|---------|---------|
| 内容缺失 | 存在未完成的占位符 | 返回相关章节补充内容 |
| 事实错误 | 数据/描述与原文不符 | 重新核对原文，修正错误 |
| 引用缺失 | 重要观点未标注来源 | 追溯原文引用位置 |
| Callout误用 | Callout类型与内容不匹配 | 重新评估并更正Callout类型 |

#### 超时处理

当CP4验证超过3次仍未通过时：
1. 输出当前版本供人工审查
2. 标注所有已知问题点
3. 暂停自动处理
4. 等待人工确认或修正

---

### CP5: 输出质量检查 (Output Quality Check)

**检查时机**: 完成自我批评循环修订后，最终输出前

#### 检查项列表 (Checklist)

- [ ] 笔记格式符合模板规范
- [ ] YAML frontmatter完整且格式正确
- [ ] Wikilink语法正确且目标存在
- [ ] Mermaid图表语法验证通过
- [ ] 图像引用路径有效
- [ ] 无乱码或格式错乱
- [ ] 4维评分已完成自评
- [ ] 质量报告已生成
- [ ] 笔记已正确存放于对应目录
- [ ] MOC索引已更新（如适用）

#### 门条件 (Gate Conditions)

所有检查项必须通过（全部打勾）。

#### 失败操作 (Failure Handling)

| 失败类型 | 具体问题 | 修复步骤 |
|---------|---------|---------|
| 格式错误 | YAML/Wikilink/Mermaid语法错误 | 启用格式修正工具逐项修复 |
| 引用失效 | 图像/链接目标不存在 | 重新验证路径或降级处理 |
| 元数据缺失 | 缺少必要frontmatter字段 | 补充缺失字段 |

#### 超时处理

当CP5验证超过3次仍未通过时：
1. 强制执行格式清理流程
2. 降级处理无法修复的问题（如图像降级为文字描述）
3. 在笔记中标注"格式待优化"
4. 仍允许输出但纳入人工复核队列

---

## 2. 自我批评循环详细规范 (Self-Criticism Loop Detailed Specification)

### 2.1 第一轮：逻辑一致性检查 (Round 1: Logical Consistency Check)

**检查重点**: 论证链条完整性、事实准确性、表述无矛盾

#### 检查清单（20项）

| 编号 | 检查项 | 通过标准 |
|------|--------|---------|
| LC1 | 背景介绍是否清晰阐述研究问题 | 问题陈述完整，包含what/why |
| LC2 | 方法描述是否可支持重复理解 | 关键步骤无遗漏 |
| LC3 | 结果呈现是否与研究问题对应 | 每个问题都有对应结果 |
| LC4 | 结论是否直接由结果导出 | 因果关系明确 |
| LC5 | 数据与结论是否匹配 | 无过度推断 |
| LC6 | 统计显著性是否正确解读 | p值/效应量使用准确 |
| LC7 | 图表数据与正文解读是否一致且互补 | 图解读中的趋势/数值与正文叙述匹配；每张图在叙述逻辑节点处嵌入而非孤立堆放 |
| LC8 | 专业术语使用是否前后一致 | 同概念同表述 |
| LC9 | 缩写首次出现是否定义 | 全文缩写有定义 |
| LC10 | 引用是否与观点对应 | 无张冠李戴 |
| LC11 | 方法优缺点是否客观陈述 | 无夸大或淡化 |
| LC12 | 局限性是否诚实承认 | 未回避不足 |
| LC13 | 不同结果是否有公平呈现 | 阳性/阴性结果同样重视 |
| LC14 | 假设前提是否明确 | 关键假设已说明 |
| LC15 | 因果与相关是否区分 | 未混淆两者 |
| LC16 | 样本描述是否充分 | n值/来源/特征明确 |
| LC17 | 重复实验是否有说明 | 技术重复/生物重复区分 |
| LC18 | 单位是否统一 | 无单位混用 |
| LC19 | 数值精度是否一致 | 同类数据精度统一 |
| LC20 | 逻辑连接词是否恰当 | 推理步骤清晰 |

---

### 2.2 第二轮：初学者可达性检查 (Round 2: Beginner Accessibility Check)

**检查重点**: 领域入门读者能否理解核心内容

#### 检查清单（15项）

| 编号 | 检查项 | 通过标准 |
|------|--------|---------|
| BA1 | 是否提供必要的领域背景 | 门外汉能理解研究 context |
| BA2 | 是否解释关键概念 | 不懂术语的读者能理解 |
| BA3 | 是否简化复杂方法描述 | 不懂算法的读者能懂原理 |
| BA4 | 图表是否有完整解读 | `[!figure]-`解读块7字段完整（目的/坐标轴/趋势/统计/对照/异常/支撑结论），前导句+后续段落连贯 |
| BA5 | 缩写在首次出现是否有全称 | 全文缩写定义完整 |
| BA6 | 是否避免不必要的行话 | 非必要专业术语已解释 |
| BA7 | 逻辑推理是否过于跳跃 | 每步推理都有衔接 |
| BA8 | 是否提供学习资源链接 | 延伸学习有路可循 |
| BA9 | 类比是否恰当 | 帮助理解而非混淆 |
| BA10 | 摘要是否可独立理解 | 读完摘要已把握全貌 |
| BA11 | 各部分过渡是否平滑 | 章节间衔接自然 |
| BA12 | 是否标注关键术语 | 重点概念已高亮/加粗 |
| BA13 | 计算步骤是否有示例 | 复杂计算有演示 |
| BA14 | 数据规模是否有参照 | 读者能感知scale |
| BA15 | 结果意义是否有通俗解释 | 非常规发现已说明价值 |

---

### 2.3 第三轮：域专家深度检查 (Round 3: Domain Expert Depth Check)

**检查重点**: 内容是否经得起领域专家审视

#### 检查清单（15项）

| 编号 | 检查项 | 通过标准 |
|------|--------|---------|
| DE1 | 方法创新点是否准确描述 | 真正的新颖之处已识别 |
| DE2 | 与现有方法关系是否清晰 | 引用正确，比较公平 |
| DE3 | 技术细节是否足够 | 同行能复现方法 |
| DE4 | 参数选择是否有依据 | 非随意设定 |
| DE5 | 基准测试是否合理 | 对比方法选择恰当 |
| DE6 | 统计分析是否规范 | 符合领域标准 |
| DE7 | 结果解读是否准确 | 无过度或不足解读 |
| DE8 | 局限性是否被同行认可 | 确实属于局限 |
| DE9 | 潜在应用是否现实 | 非画饼 |
| DE10 | 未来方向是否有价值 | 确实值得探索 |
| DE11 | 引用是否覆盖关键文献 | 无重大遗漏 |
| DE12 | 领域术语使用是否精确 | 无歧义 |
| DE13 | 实验设计是否合理 | 控制变量适当 |
| DE14 | 数据质量控制是否充分 | 质量保障到位 |
| DE15 | 结论强度是否适当 | 与证据匹配 |

---

### 2.4 图文融合专项检查 (Figure-Text Integration Check)

**检查时机**: CP4内容质量检查阶段，与自我批评循环并行执行

#### 检查清单（10项）

| 编号 | 检查项 | 通过标准 |
|------|--------|---------|
| FT1 | 是否有图被孤立堆放在"图像"小节下 | 无任何图孤立于叙述之外，所有图在叙述逻辑节点处嵌入 |
| FT2 | 每张图是否有前导句 | 前导句明确预告图展示什么（1句话） |
| FT3 | 每张图是否有 `[!figure]-` 解读块 | Callout格式正确且非空，默认折叠 |
| FT4 | 解读块7字段是否完整 | 目的/坐标轴/趋势/统计/对照/异常/支撑结论均存在 |
| FT5 | 数据图解读是否含趋势+统计 | 具体趋势方向+具体p值/效应量 |
| FT6 | 表格图解读是否含关键数值复述 | 至少复述3个关键数值（便于搜索索引） |
| FT7 | 公式截图解读是否含符号定义 | 每个主要符号有中文含义说明 |
| FT8 | 流程图解读是否含决策节点 | 明确指出流程中的关键分支/决策点 |
| FT9 | 解读最后一句是否连接论文结论 | 明确写"图中XX特征支撑论文XX结论" |
| FT10 | 图的插入位置是否匹配叙述逻辑 | 不在方法小节插入纯结果图，不在结果小节插入纯方法图 |

#### 门条件

至少8/10项通过，其中FT1（无孤立图）和FT4（7字段完整）必须通过。

### 2.5 修订项格式与处理

发现问题时，按以下格式记录：

```markdown
## 修订项记录

### 修订项 #{序号}

| 字段 | 内容 |
|------|------|
| **位置** | {在笔记中的具体位置，如"3.2节第2段"} |
| **问题描述** | {具体问题的准确描述} |
| **严重程度** | {critical / major / minor} |
| **建议修改** | {具体可行的修改建议} |
| **原文摘录** | {问题所在的原文片段} |
| **修订后文本** | {修改后的文本（如已修订）} |

---

### 2.5 循环退出条件

满足以下任一条件可退出循环：

1. **理想退出**: 三轮检查均通过（critical=0, major<=2, minor<=5）
2. **次优退出**: 剩余修订项均为minor且数量<=3
3. **强制退出**: 已完成3轮检查，虽有剩余问题但已达处理极限

### 2.6 严重程度定义

| 级别 | 符号 | 定义 | 处理要求 |
|------|------|------|---------|
| Critical | 🔴 | 事实错误/逻辑断裂/严重误导 | 必须修复 |
| Major | 🟡 | 重要遗漏/表述不清/影响理解 | 应修复 |
| Minor | 🔵 | 格式问题/表述优化/风格建议 | 建议修复 |

## 3. 四维评分标准 (Four-Dimensional Scoring Standard)

### 3.1 可复现性 (Reproducibility) — 5级描述

| 分数 | 等级 | 描述 |
|------|------|------|
| 1 | 不合格 | 方法描述完全缺失或极度简略，无法提取任何可操作信息 |
| 2 | 不足 | 方法描述存在但不完整，缺少关键步骤，无法独立复现 |
| 3 | 基本合格 | 方法描述完整度>70%，关键步骤已说明，但参数/条件有缺失 |
| 4 | 良好 | 方法描述详细，可支持复现，参数完整，仅细微遗漏 |
| 5 | 优秀 | 方法描述极其详尽，含参数优化空间、常见问题处理、代码/命令完整 |

### 3.2 清晰度 (Clarity) — 5级描述

| 分数 | 等级 | 描述 |
|------|------|------|
| 1 | 不合格 | 结构混乱，表述歧义多，读者无法把握核心信息 |
| 2 | 不足 | 结构松散，部分表述不清，读者需反复阅读才能理解 |
| 3 | 基本合格 | 结构合理，主要内容清晰，但细节处仍有歧义 |
| 4 | 良好 | 表述清晰，逻辑流畅，图表配合好，易于理解 |
| 5 | 优秀 | 极其清晰，图文并茂，专业术语适度解释，初学者也可理解 |

### 3.3 完整性 (Completeness) — 5级描述

| 分数 | 等级 | 描述 |
|------|------|------|
| 1 | 不合格 | 核心模块缺失（背景/方法/结果/结论不完整） |
| 2 | 不足 | 核心模块存在但内容严重不足，关键信息遗漏>30% |
| 3 | 基本合格 | 核心模块完整，关键信息遗漏<15%，但有重要细节缺失 |
| 4 | 良好 | 模块完整，信息完整度高，仅边缘信息有缺失 |
| 5 | 优秀 | 信息全面完整，无重要遗漏，含补充信息和延伸资源 |

### 3.4 深度 (Depth) — 5级描述

| 分数 | 等级 | 描述 |
|------|------|------|
| 1 | 不合格 | 仅表面描述，无任何深度分析 |
| 2 | 不足 | 有一定分析但流于表面，未触及本质 |
| 3 | 基本合格 | 分析深度适中，能抓住主要特点，但未深入机制 |
| 4 | 良好 | 分析深入，能揭示内在规律，有独到见解 |
| 5 | 优秀 | 分析极其深刻，见解独到，能引发新的思考 |

### 3.5 及格线与评分报告格式

**及格条件**: 均分 >= 3.5，且无单项 < 3

**评分报告格式**:

```markdown
## 4维评分报告

| 维度 | 分数 | 等级 | 说明 |
|------|------|------|------|
| 可复现性 | X/5 | [等级] | [1-2句说明] |
| 清晰度 | X/5 | [等级] | [1-2句说明] |
| 完整性 | X/5 | [等级] | [1-2句说明] |
| 深度 | X/5 | [等级] | [1-2句说明] |
| **综合** | **X/5** | [总等级] | [总体评价] |

### 及格判定

- 综合均分: X/5 {"">=3.5? "✅ 及格" : "❌ 不及格"}
- 最低单项: X/5 {"">=3? "✅ 达标" : "❌ 存在单项不合格"}

**最终结论**: {"">=3.5 && 最低>=3 ? "通过质量标准" : "需修订后重审"}
```
