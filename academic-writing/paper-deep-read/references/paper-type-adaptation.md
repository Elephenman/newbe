# 论文类型适配矩阵 (Paper Type Adaptation Matrix) — v4.0

> 本文档定义了论文深度解读技能中针对不同论文类型的适配策略，包括分类决策树、六种论文类型的详细配置，以及混合类型处理方法。
> 
> **v4.0核心变更**：所有Callout类型统一为4+1体系（abstract/figure/critique/tip/question），Figure解读默认叙述段落嵌入。

---

## 1. 类型分类决策树 (Type Classification Decision Tree)

使用以下是的/否问题序列来引导论文类型分类：

```
开始分类
│
├── 论文是否包含原创实验数据？
│   ├── 否 → 进入下一步
│   │   └── 是否系统梳理某领域已有研究？
│   │       ├── 是 → 综述论文 (Review)
│   │       └── 否 → 是否提供新工具/数据库/资源？
│   │               ├── 是 → 数据库/资源论文 (Database/Resource)
│   │               └── 否 → 分类失败，需人工判定
│   │
│   └── 是 → 进入下一步
│       └── 方法部分占比是否 >40%？
│           ├── 是 → 方法学论文 (Methodological)
│           └── 否 → 进入下一步
│               └── 是否提出新工具/数据库/资源？
│                   ├── 是 → 数据库/资源论文 (Database/Resource)
│                   └── 否 → 进入下一步
│                       └── 结果部分是否 >50%？
│                           ├── 是 → 发现型论文 (Discovery)
│                           └── 否 → 进入下一步
│                               └── 是否包含命令行/代码/计算流水线？
│                                   ├── 是 → 计算/生信论文 (Computational/Bioinformatics)
│                                   └── 否 → 进入下一步
│                                       └── 是否包含PDB/冷冻电镜/结构解析？
│                                           ├── 是 → 结构生物学论文 (Structural Biology)
│                                           └── 否 → 混合特征，按主要特征归类
```

### 决策树关键节点说明

| 节点编号 | 问题 | 阈值 | 说明 |
|---------|------|------|------|
| Q1 | 是否包含原创实验数据 | — | 区分原创研究与综述类文献 |
| Q2 | 方法部分占比 | >40% | 基于PDF结构或章节长度估算 |
| Q3 | 是否提出新工具/数据库/资源 | — | 工具指软件/算法，资源指数据集 |
| Q4 | 结果部分占比 | >50% | 评估结果章节在全文中的权重 |
| Q5 | 是否包含命令行/代码/计算流水线 | — | 指生信/计算分析类内容 |
| Q6 | 是否包含PDB/冷冻电镜/结构解析 | — | 专指结构生物学实验方法 |

### 分类优先级规则

当论文同时满足多个类型条件时，按以下优先级归类：

1. **结构生物学论文** — 最高优先级，因其结构数据具有明确标识
2. **计算/生信论文** — 特征明显（代码/命令行）
3. **数据库/资源论文** — 工具/资源型论文
4. **方法学论文** — 方法部分突出
5. **发现型论文** — 结果驱动型
6. **综述论文** — 最低优先级，需无原创数据

## 2. 六种论文类型详细配置 (Six Paper Types Detailed Configuration)

### 2.1 方法学论文 (Methodological Paper)

| 配置项 | 值 |
|--------|-----|
| **识别特征** | 方法章节占全文30%-60%；提出新算法/模型/框架；包含基准测试或与已有方法对比；方法创新点是核心贡献 |
| **重点模块** | 方法原理详解、算法伪代码、参数设置、实验设计、对比方法描述 |
| **弱化模块** | 详细结果描述（聚焦方法有效性即可）、扩展应用讨论 |
| **输出权重分布** | 背景15%, **方法45%**, 结果20%, 结论15%, 其他5% |
| **Callout使用模式** | `[!tip]-` 方法优势说明；`[!critique]-` 方法脆弱性攻击（仅真正硬伤）；`[!figure]-` 仅Supplementary/表格型 |
| **常见陷阱** | 将方法描述等同于方法原理复述；忽视消融实验分析；忽略计算复杂度分析；方法对比不公平 |
| **QA重点检查** | 方法原理是否足够详细以支持复现；参数设置是否有依据；对比基线是否合适；实验设置是否公平 |
| **批判评价必含维度** | 6.1(学术缺口), 6.3(假设判决), 6.4(攻击清单), 6.7(核心贡献) |
| **批判评价可选维度** | 6.2(理论DNA), 6.5(边界条件), 6.6(范式地图), 6.8(研究裂变) |

---

### 2.2 发现型论文 (Discovery Paper)

| 配置项 | 值 |
|--------|-----|
| **识别特征** | 结果章节占全文50%以上；发现是核心贡献；通常包含多角度验证；数据驱动的洞察 |
| **重点模块** | 结果呈现的逻辑链条、关键发现提炼、发现的意义阐释、结果的可视化呈现 |
| **弱化模块** | 方法详细描述（可简化至简要流程）、方法学创新评估 |
| **输出权重分布** | 背景15%, 方法20%, **结果45%**, 结论15%, 其他5% |
| **Callout使用模式** | `[!abstract]-` 关键发现标注；`[!critique]-` 重要结论突出（仅真正硬伤）；`[!tip]-` 实践要点 |
| **常见陷阱** | 发现陈述过于平淡；忽视阴性结果；过度解读有限数据；验证逻辑链断裂 |
| **QA重点检查** | 发现是否有充分数据支持；多角度验证是否完整；结论是否在数据支持范围内；是否诚实地讨论了局限性 |
| **批判评价必含维度** | 6.1(学术缺口), 6.3(假设判决), 6.5(边界条件), 6.7(核心贡献) |
| **批判评价可选维度** | 6.2(理论DNA), 6.4(攻击清单), 6.6(范式地图), 6.8(研究裂变) |

---

### 2.3 综述论文 (Review Paper)

| 配置项 | 值 |
|--------|-----|
| **识别特征** | 无原创实验数据；系统梳理某一领域研究进展；通常按主题/时间/方法分类；引用文献数量>50篇 |
| **重点模块** | 领域全景图谱、关键进展时间线、各方法/学派对比、争议点与共识、未来方向 |
| **弱化模块** | 单一方法的详细原理、具体实验流程 |
| **输出权重分布** | **背景30%**, 方法15%, 结果25%, 结论25%, 其他5% |
| **Callout使用模式** | `[!tip]-` 实践建议；`[!question]-` 思考题/研究裂变；`[!abstract]-` 关键结论 |
| **常见陷阱** | 文献覆盖不全面；分类体系混乱；缺乏critical analysis；只描述不评价 |
| **QA重点检查** | 文献覆盖是否系统；分类是否合理；是否有作者独立见解；争议点是否公正呈现 |
| **批判评价必含维度** | 6.2(理论DNA), 6.6(范式地图), 6.7(核心贡献) |
| **批判评价可选维度** | 6.1(学术缺口), 6.3(假设判决), 6.4(攻击清单), 6.5(边界条件), 6.8(研究裂变) |

---

### 2.4 数据库/资源论文 (Database/Resource Paper)

| 配置项 | 值 |
|--------|-----|
| **识别特征** | 论文描述新数据库/工具/数据集；包含网站/服务器介绍；数据收集/整合是核心贡献；通常有在线访问入口 |
| **重点模块** | 资源描述与访问方式、数据收集/处理流程、数据规模与覆盖范围、用户使用案例、与已有资源对比 |
| **弱化模块** | 深度算法原理（可简化）、详细实验参数 |
| **输出权重分布** | 背景20%, 方法25%, 结果30%, 结论20%, 其他5% |
| **Callout使用模式** | `[!tip]-` 访问地址标注+使用方法；`[!critique]-` 数据质量局限（仅真正硬伤）；`[!abstract]-` 数据规模强调 |
| **常见陷阱** | 工具使用方法描述不清；数据来源不透明；与其他资源对比不客观；网站可持续性未说明 |
| **QA重点检查** | 资源是否可访问；使用方法是否清晰；数据质量控制是否充分；与现有资源关系是否明确 |
| **批判评价必含维度** | 6.1(学术缺口), 6.5(边界条件), 6.7(核心贡献) |
| **批判评价可选维度** | 6.2(理论DNA), 6.3(假设判决), 6.4(攻击清单), 6.6(范式地图), 6.8(研究裂变) |

### 2.5 计算/生信论文 (Computational/Bioinformatics Paper)

| 配置项 | 值 |
|--------|-----|
| **识别特征** | 包含代码/命令行/计算流程；生物信息学分析为主线；通常有GitHub仓库；benchmark评估是标准配置 |
| **重点模块** | 计算流程与依赖环境、代码可复现性、参数优化分析、性能基准测试、与现有流程对比 |
| **弱化模块** | 湿实验详细描述、生物意义深度挖掘 |
| **输出权重分布** | 背景15%, **方法40%**, 结果25%, 结论15%, 其他5% |

| 配置项 | 值 |
|--------|-----|
| **识别特征** | 包含PDB ID或结构数据；冷冻电镜/X射线晶体学方法；结构解析是核心贡献；通常讨论结构-功能关系 |
| **重点模块** | 结构解析方法与参数、结构特征描述、活性位点/结合口袋分析、结构比较、与已知结构对比 |
| **弱化模块** | 常规生化实验描述、一般性功能验证 |

| **Callout使用模式** | `[!tip]-` PDB ID标注+结构分析要点；`[!critique]-` 分辨率/质量限制（仅真正硬伤）；`[!figure]-` 仅Supplementary |

| **QA重点检查** | 结构质量是否达标；功能推断是否有实验支持；结构比较是否准确；数据 deposition 是否完整 |

## 3. 混合类型处理策略 (Mixed Type Handling Strategy)

### 3.1 混合类型识别

当论文同时具备多种类型特征时，需进行混合类型标注。常见混合类型组合：

| 组合类型 | 识别特征 | 主类型判定规则 |
|---------|---------|--------------|
| 方法学 + 计算/生信 | 新方法含代码实现 | 两者并重，权重叠加 |
| 方法学 + 发现型 | 新方法 + 重大发现 | 发现为主，方法为辅 |
| 数据库 + 计算/生信 | 新资源含分析流程 | 资源为主，计算为工具 |
| 发现型 + 结构生物学 | 新发现含结构证据 | 发现为主，结构为证据 |
| 综述 + 数据库 | 系统综述 + 新资源 | 综述为主，资源为补充 |

### 3.2 主类型+副类型标注

采用 `主类型[副类型]` 格式标注：

```
## 论文类型: 发现型[结构生物学]

本文档将{论文标题}归类为**发现型论文**，同时具有结构生物学特征。
- 主类型特征: 结果驱动型发现，占比60%
- 副类型特征: 关键结论依赖结构证据，占比40%
```

### 3.3 模块权重混合计算

当论文为混合类型时，按以下公式计算模块权重：

```
最终权重 = 主类型权重 × 0.6 + 副类型权重 × 0.4
```

#### 计算示例：方法学[计算/生信]

| 模块 | 方法学权重 | 计算/生信权重 | 最终权重 |
|------|-----------|--------------|---------|
| 背景 | 15% | 15% | 15.0% |
| 方法 | 45% | 40% | 43.0% |
| 结果 | 20% | 25% | 22.0% |
| 结论 | 15% | 15% | 15.0% |
| 其他 | 5% | 5% | 5.0% |

### 3.4 特殊模块激活规则

某些混合类型组合需要激活额外的分析模块：

| 混合类型组合 | 激活的特殊模块 | 输出要求 |
|------------|--------------|---------|
| 方法学 + 计算/生信 | 命令行重现 + 参数优化分析 | 完整可执行命令序列 + 敏感性分析图 |
| 方法学 + 发现型 | 方法有效性验证 + 机制洞察 | 消融实验解读 + 潜在机制的讨论 |
| 数据库 + 计算/生信 | 使用案例分析 + 性能基准 | 典型场景演示 + 吞吐量/准确性对比 |
| 发现型 + 结构生物学 | 结构-功能映射 + 口袋分析 | 关键残基标注 + 结合能力评估 |
| 综述 + 数据库 | 资源生态图 + 使用建议 | 资源关系图 + 按场景推荐表 |
| 方法学 + 结构生物学 | 结构基础验证 + 机制解释 | 结构支撑的方法验证 + 功能推断 |

## 4. 批判性评价维度适配

每种论文类型对批判性评价8个维度的要求不同。详细规格参照 `references/critical-analysis-guide.md`。

### 4.1 必含/可选矩阵

| 论文类型 | 必含维度 | 可选维度 |
|----------|---------|---------|
| 方法学 | 6.1, 6.3, 6.4, 6.7 | 6.2, 6.5, 6.6, 6.8 |
| 发现型 | 6.1, 6.3, 6.5, 6.7 | 6.2, 6.4, 6.6, 6.8 |
| 综述 | 6.2, 6.6, 6.7 | 6.1, 6.3, 6.4, 6.5, 6.8 |
| 数据库/资源 | 6.1, 6.5, 6.7 | 6.2, 6.3, 6.4, 6.6, 6.8 |
| 计算/生信 | 6.1, 6.3, 6.4, 6.7 | 6.2, 6.5, 6.6, 6.8 |
| 结构生物学 | 6.1, 6.3, 6.5, 6.7 | 6.2, 6.4, 6.6, 6.8 |

### 4.2 Callout类型分配（统一4+1）

所有批判性评价维度统一使用4+1 Callout体系：

| 维度 | Callout类型 | 适用论文类型 |
|------|------------|------------|
| 6.1 学术缺口 | `[!critique]-` | 全类型必含 |
| 6.2 理论DNA | `[!abstract]-` | 综述必含，其他可选 |
| 6.3 假设判决 | `[!critique]-` | 方法学/发现/生信/结构必含 |
| 6.4 攻击清单 | `[!critique]-` | 方法学/生信必含，其他可选（仅真正硬伤） |
| 6.5 边界条件 | `[!critique]-` | 发现/数据库/结构必含 |
| 6.6 范式地图 | 表格+Mermaid | 综述必含，其他可选 |
| 6.7 核心贡献 | `[!critique]-` | 全类型必含 |
| 6.8 研究裂变 | `[!question]-` | 全类型可选(推荐) |

**通用方法局限**：不使用Callout，使用🧪引用块+[[4.X]]链接

## 附录：类型快速识别清单

### 方法学论文快速识别

- [ ] 标题含 "method", "approach", "framework", "model", "algorithm"
- [ ] 摘要以 "we propose/present/develop" 开头
- [ ] 方法章节独立成篇且篇幅较长
- [ ] 有与现有方法的对比实验

### 发现型论文快速识别

- [ ] 标题描述一个现象/规律/机制
- [ ] 结果章节包含多个子章节分别展示不同角度证据
- [ ] 图表数量多且信息密度高
- [ ] 讨论章节深入分析发现的意义

### 综述论文快速识别

- [ ] 标题含 "review", "survey", "overview", "landscape"
- [ ] 引用文献数量>50
- [ ] 无 "materials and methods" 或 "experiments" 章节
- [ ] 章节按主题或时间组织而非按结果呈现

### 数据库/资源论文快速识别

- [ ] 标题含数据库/服务器/资源名称
- [ ] 摘要明确提供访问URL
- [ ] 包含数据规模统计（条目数、物种覆盖等）
- [ ] 有 "user guide" 或 "tutorial" 类章节

### 计算/生信论文快速识别

- [ ] 包含代码仓库链接（GitHub等）
- [ ] 方法章节包含软件/工具名称及版本
- [ ] 有计算性能基准测试
- [ ] 涉及序列分析、结构预测等功能

### 结构生物学论文快速识别

- [ ] 包含PDB ID（格式：4字符）
- [ ] 方法章节描述晶体学或冷冻电镜流程
- [ ] 包含 electron density map 或 structure alignment
- [ ] 分辨率（resolution）作为质量指标出现
