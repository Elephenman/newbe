# 解读写作规范 — v4.1

## 核心原则：解读为主，批判为辅

本Skill的初心是让小白通过解读报告学会论文思路、写作方法和复现。
因此解读是核心产出，批判是补充视角。

## 正文量规则

1. 每个实验段的正文解读 ≥ 5-8句
2. 正文解读句数 ≥ 2× 批判句数
3. 通用方法局限不单独开Callout，用🧪引用块

## ⚠️ 最高优先级：原文caption对照（v4.1新增）

**这是解读写作最核心的规则。违反此规则会导致解读与论文图片完全不对应，这是不可接受的硬伤。**

### 规则
1. **Panel编号和子标题必须严格按论文原文Figure caption**
   - 从论文Results或Figure legends中定位该Figure的完整caption
   - 提取所有Panel编号(A/B/C/...)和描述 → 列出完整Panel清单
   - 绝不允许凭想象编造Panel编号或内容
   - 论文有7个Panel(A-G) → 解读必须包含全部7个Panel

2. **斜体caption必须引用原文Figure caption**
   - 格式：`*图N {{原文Figure caption完整标题}}。(A) {{原文Panel A描述}}；(B) {{原文Panel B描述}}。*`
   - 不是用自己的话概括，而是从论文原文提取
   - 可适当精简但Panel编号和核心描述必须保留原文措辞

3. **图片文件名映射必须正确**
   - fig1.png对应论文Figure 1，fig2.png对应Figure 2
   - 不同Section可引用同一张图片文件
   - 绝不允许引用不存在的图片文件

### 为什么这很重要
论文Figure是科学论证的核心视觉证据。如果Panel编号和内容与论文不对应，读者无法将解读与图片对照理解，解读失去了与论文原文的锚定关系，变成虚构文本。

## 视觉层级规则

| 类型 | 格式 | 语义 | 示例 |
|------|------|------|------|
| **结构标签** | `**标签：**`冒号结尾 | 告诉读者"接下来是什么内容" | **实验目的：** **核心发现：** **逻辑衔接：** |
| **Panel标签** | `**Panel X — 子标题**`破折号连接 | 告诉读者"这是图中哪个面板的解读" | **Panel A — MEME motif logo** |
| **斜体caption** | `*图N 标题。(a)...(b)...*` | 图片标题，在Panel解读之前 | *图1 DdaA结合motif鉴定。(a)...；* |

**视觉区分规则**：
- 结构标签 → `**标签：**`（冒号结尾，后面是正文段落）
- Panel标签 → `**Panel X — 子标题**`（破折号连接，后面是2-3句解读）
- 斜体caption → `*图N 标题。(a)...(b)...*`（斜体，在Panel标签之前）

## 逻辑衔接（替代推导语+策略解释）

推导语 = 简单预告"接下来作者做了XX"
逻辑衔接 = 解释完整的"发现→追问→设计→预期→验证"闭环

### ★转折点：完整5步闭环

```
**逻辑衔接：** 3.X的pulldown在非交联条件下捕获了DdaA，但无法排除间接结合。
→ 追问：DdaA是否直接结合motif DNA？
→ 设计：EMSA是回答这个问题的最直接方法（[[4.2 EMSA Protocol]])——纯化蛋白+纯化DNA，无中介因子。
→ 预期：如果DdaA直接结合，EMSA中应出现阻滞带；如果不结合（间接结合），则无阻滞。
→ 验证：3.2的EMSA结果确认了前者——DdaA与motif DNA的结合呈浓度依赖性阻滞带。
```

### （确认点）：简洁3步

```
**逻辑衔接：** 进化分析确认DdaA在402个Moraxellaceae基因组中普遍存在 → 下一步验证DdaA的DDR调控功能（3.5）。
```

### ★双重节点：5步闭环+双预期

```
**逻辑衔接：** ChAP-seq验证了DdaA对DDR基因的调控（确认3.1-3.2结论），但发现了DdaA也结合CRISPR-Cas和RM系统启动子 →
追问DdaA是否同时调控抗噬菌体防御 →
预期1（如果DdaA调控抗噬菌体）：敲除株应噬菌体敏感；预期2（如果不调控）：噬菌体抵抗力不变 →
验证见3.5-3.8。
```

### 转折/确认/双重判断规则

| 标记 | 语义 | 叙述量 | 逻辑衔接格式 |
|------|------|--------|-------------|
| ★转折点 | 实验结论推翻前假设或揭示新方向 | ≥8句 | 5步闭环 |
| （确认点） | 实验仅验证已知结论 | 4-6句 | 3步简洁 |
| ★双重节点 | 既验证已知又揭示新方向 | ≥8句 | 5步闭环+双预期 |

**生成规则**：如果一个实验同时满足"确认前结论"和"揭示新发现"，标记为★双重节点，并在逻辑衔接中明确标注"确认XX+揭示YY"。

## Figure解读方式

默认：叙述段落嵌入（不用Callout折叠）
→ 图片放在叙述流中
→ **Panel X — {{子标题}}** 加粗标签引导
→ 每Panel 2-3句解读直接写在标签后
→ **图的整体逻辑：** 1句概括各Panel间关系

排版效果：一段文字 → 图片 → caption(斜体) → Panel解读 → 继续文字
→ 像论文本身的排版一样自然流畅
→ 读者不需要"展开→阅读→折叠→继续"反复操作

例外（使用[!figure]- Callout折叠）：
→ Supplementary Figure（非核心补充图）
→ 数据表格型Figure（不是面板型复合图）

## 发现叙述 vs 发现表格

优先用叙述段落而非表格呈现发现：
- 叙述段落能展开数据细节和含义
- 表格只适合速查，不适合学习理解
- 发现表可以保留在Section 8速查区

## 批判的三种表达形式

1. **🧪引用块**：通用方法局限（含[[4.X]]链接）
   ```markdown
   > 🧪 方法局限：DNA pulldown在非交联条件下进行，可能遗漏间接结合位点。交联验证参见[[4.1 DNA pulldown Protocol]]。
   ```
2. **[!critique]- Callout**：真正的方法论硬伤（如统计不透明、单时间点无法判断先后）
3. **Section 6汇总**：假设判决、攻击清单(仅真正硬伤)、边界条件、范式贡献、研究裂变

## 分隔规则

- ★转折点/★双重节点 → 上方有一条`---`
- （确认点） → 上方无`---`
- Section级别 → 保留`---`
- 规则：全文只有3-4条分隔线（只标记论证阶段转换），而不是每段都有
